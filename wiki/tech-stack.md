# Tech stack

The stack as shipped in the Wainwrights MVP. Versions are the majors in
`package.json`; the full decision record, with fallbacks and the research
behind each choice, is in the
[implementation plan](implementation-plan.md#verified-technology-decisions).

## Application

- **React 19** + **Vite 8** single-page app — no server, no accounts.
- **TypeScript 6** in strict mode; `tsc -b --noEmit` runs as its own check.
- **Tailwind CSS 4** via the `@tailwindcss/vite` plugin, with the design
  tokens defined in a CSS `@theme` block (see [Design](design.md)).
- **Routing** is a small hand-rolled hash router (`#/`, `#/tracker`,
  `#/data`, `#/settings`) — four pages do not need a routing library, and
  hash routes avoid SPA-404 handling on static hosting.

## Map

- **MapLibre GL JS 5** via `@vis.gl/react-maplibre` 8. Runtime MapLibre
  imports stay inside `src/map/`: `MapView.tsx` is the only module that
  imports the React wrapper, and `terrain.ts` imports `maplibre-gl` directly
  only to register the shared DEM protocol. Modules elsewhere may import
  MapLibre types only.
- **Basemap**: a committed fork of OpenFreeMap's Dark style, with vector
  tiles from the public OpenFreeMap instance — free, key-free, no request
  limits. The tile URL is isolated in `src/map/config.ts` so it could be
  swapped for a self-hosted source.
- **Terrain**: AWS Terrain Tiles (Terrarium format, public S3, key-free) as
  a `raster-dem` hillshade, plus `maplibre-contour` generating contour lines
  in the browser. Terrain is a user preference, off or on per device.

## State and data

- **Zustand 5** with the `persist` middleware — progress in
  `munro.progress.v1` and preferences in `munro.prefs.v1`, both in
  localStorage, both versioned with `migrate`.
- **Zod 4** (`zod/mini`) validates peak data in tests and backup imports at
  runtime.
- Peak and boundary data are reviewed static JSON files generated by the
  scripts in `scripts/` — see [Data](data.md).
- **Image export** captures the MapLibre canvas and composites the title,
  stats and attribution on an offscreen 2D canvas. The export engine is a
  lazily loaded chunk so it stays out of the initial bundle.

## Tooling

- **Vitest 4** + jsdom + React Testing Library for unit and component
  tests, with `@vitest/coverage-v8` thresholds.
- **Playwright 1** end-to-end tests against the production build, in a
  desktop Chromium project and an iPhone-13-sized mobile project.
- **ESLint 10** (flat config, `strictTypeChecked`) and **Prettier 3** with
  the Tailwind class sorter; husky + lint-staged on commit.
- **GitHub Actions** run checks, build, e2e, docs and CodeQL on every PR;
  merges to `main` deploy to <https://munro.welbournesecurity.com>, and each
  PR gets its own preview. See [Operations](operations.md).
- **Node 24** (`.nvmrc` and `engines`).

## Architectural rules

- **Local-first**: no backend, no accounts, no authentication in the MVP.
  User progress lives in the browser with JSON import/export as backup.
- **Key-free**: no API keys, tokens or secrets anywhere in code, config or
  workflows.
- **Data separated from progress**: peak source data and user bagging
  records are distinct schemas — see [Data](data.md).
- **Lists are data, not code**: adding another hill list must be a
  straightforward data change, not a refactor.
- **Pure domain**: `src/domain/` has no React and no MapLibre imports;
  runtime MapLibre imports stay inside `src/map/` (other modules may use
  MapLibre types only).
- **Web-first, multi-platform-ready**: the stack targets the browser first,
  but keeps the iPhone and Android paths open — see
  [Platforms](platforms.md).
